# Leslie — History

## Project Context

- **Project:** Browser-based roguelike dungeon crawler
- **Stack:** Vanilla JavaScript, HTML5 Canvas, no bundler — modules export via `window.X` globals
- **Owner:** Nabeel
- **Joined:** 2026-02-26

## Key Files

- `src/core/constants.js` — Tile types, classes (Warrior/Mage/Rogue/Cleric), item types, XP table, regen rates
- `src/core/utils.js` — Seeded PRNG, distance calcs, grid helpers
- `src/core/gameState.js` — State singleton, entity/item factories
- `src/dungeon/generator.js` — BSP dungeon generation, 50x50 grids
- `src/systems/combat.js` — Turn-based combat, 12 abilities, status effects, class-based regen
- `src/systems/ai.js` — A* pathfinding, 5 AI behaviors (aggressive, flanking, cautious, ranged, boss)
- `src/entities/monsters.js` — 13 monster types across 10 floors
- `src/items/items.js` — 50+ items, identification system, loot tables, equipment stat application
- `src/rendering/renderer.js` — Canvas tilemap, fog of war, screen shake, damage numbers
- `src/ui/hud.js` — HUD, minimap, title/death/victory/help/inventory screens
- `src/main.js` — Game loop, input handling, save/load, permadeath

## Learnings

- **Double XP / double level-up bug**: `main.js:tryMove()` awards XP via `player.xp += (target.level || 1) * 10` AND `CombatSystem.meleeAttack()` → `onKill()` awards XP via `victim.xpValue`. Both paths fire on kill = double XP. Additionally, `main.js:checkLevelUp()` and `combat.js:checkLevelUp()` have different stat formulas — both will run, causing double level-up with inconsistent stat gains.
- **Save/load loses all non-schema fields**: JSON round-trip drops `statusEffects`, `tags`, `xpValue`, `templateKey`, `_buffs`, `_defKey`, `_enraged`, `_telegraphing`, `_summonedPhase2/3`, identification state (`_idMap`, `_identifiedKeys`). Loaded games will have broken monsters, broken items, broken boss AI.
- **ItemSystem.init() never called**: `startNewGame()` in `main.js` never calls `ItemSystem.init(rng)`. The identification maps are never populated, so all potions/scrolls will have undefined appearance names. Decision doc explicitly says this must be called.
- **ItemSystem.dropLoot() never called**: `combat.js:onKill()` never calls `ItemSystem.dropLoot()`. Monsters never drop loot on death despite the system existing.
- **ItemSystem.tickBuffs() never called**: Neither `main.js` nor `ai.js` calls `ItemSystem.tickBuffs()` for any entity. Buff/debuff timers from potions, scrolls, and food never tick down — they last forever.
- **Math.random() used in deterministic systems**: `combat.js:calcBaseDamage()` and multiple AI behavior functions use `Math.random()` instead of seeded RNG. This breaks determinism and makes save/load replays inconsistent.
- **ai.js loads before monsters.js**: Script order in `index.html` has `ai.js` before `monsters.js`, but `ai.js:summonMinions()` calls `window.MonsterFactory.createMonster()`. Works because it's called at runtime not load-time, but fragile.
- **No inventory size limit**: Players can hoard unlimited items. No weight, no cap.
- **Combat phase threshold (distance ≤ 2) is too tight** for ranged enemies that can hit from 6+ tiles away — player gets attacked without the COMBAT indicator.
- **Helmet and amulet slots have no TYPE_WEIGHTS entry**: TYPE_WEIGHTS only includes weapon/armor/potion/scroll/ring/food. Helmets, boots, and amulets can only appear via the 30% bonus roll, making them extremely rare.
- **Previous 6 critical fixes all landed correctly** (Decision #13): Double XP, createItem custom props, init(), dropLoot(), tickBuffs(), save/load fields. No regressions. Dead fallback code in main.js remains (lines 210-219, 395-406) but is harmless.
- **Player combat status effects never tick** (NEW BUG, 2026-02-27): `CombatSystem.tickStatusEffects(player)` and `CombatSystem.processTurnStart(player)` are never called anywhere. Only monsters get their combat effects ticked (via `processMonsterTurn` → `processTurnStart`). Consequence: player immune to poison/bleed DOTs, War Cry/Divine Shield buffs permanent and stackable, player can never be stunned. This is worse than the original 6 criticals because it breaks combat balance in BOTH directions.
- **DOT kills on monsters award no XP/loot**: When `tickStatusEffects()` kills a monster via poison or bleed, it sets `alive = false` but never calls `onKill()`. Players using DOT strategies lose all rewards.
- **Scroll of Fireball kills bypass onKill()**: `items.js:_aoeFireball()` directly sets `alive = false` without routing through CombatSystem. Same consequence — no XP, no loot.
- **Sheldon's "duplicate checkLevelUp" is dead code**: The fallback path in `tryMove()` only fires when `window.CombatSystem` doesn't exist. Since combat.js always loads before main.js, this code is unreachable. Sheldon overrated it as 🔴 CRITICAL — it's 🟡 SERIOUS (maintenance hazard).
- **Save/load split-brain player is real**: After JSON deserialization, `state.player` and the player entry in `state.entities` are separate objects. Movement updates one, entity queries search the other. AoE attacks target stale player position. Confirmed Sheldon's finding.
- **Wait-to-win regen loop is the #1 fun-killer**: No food clock or urgency mechanic means optimal play is always wait-to-full-heal between every fight. Removes all tension.
- **Retrospective (2026-02-27)**: Participated in retrospective ceremony. Key findings: (1) All 9+ critical bugs were integration gaps — modules that never called each other — not logic bugs. (2) 85-test suite had zero cross-module integration tests, creating false confidence. (3) `window.X && X.method` graceful degradation guards hid integration failures by turning missing calls into silent no-ops. (4) Parallel development of 5 modules without integration checkpoints was the root cause. (5) Issued 5 non-negotiable process demands: integration tests before "done," no graceful degradation in dev mode, Critic reviews before completion, decision docs as checklists, and 20% minimum integration test coverage. Filed to `.squad/decisions/inbox/leslie-retrospective.md`.
- **Design Review (2026-02-27)**: Full-game critic review. Key findings: (1) Regen cooldown is inverted — starts at 0, counts DOWN, so player gets zero regen at game start until first combat ends (`gameState.js:81`, `combat.js:625-627`). (2) `spawnDamageNumber`/`triggerShake` are never called from combat — visual feedback is dead code (`renderer.js:62-90`). (3) Wait-to-win remains #1 fun-killer since regen cooldown only limits turns, not total recovery. (4) No hunger/food clock = zero exploration pressure. (5) Score formula punishes turns but regen cooldown incentivizes waiting = contradictory design. (6) Auto-ranged attack on movement is invisible to player — no feedback, fires into fog, confusing. (7) Mage regen cooldown 8 turns is a band-aid — Arcane Shield 30-absorb + 3 mana/turn regen still makes Mage unkillable with patience.
- **Design Review #2 (2026-02-27)**: Follow-up review after team addressed top 3 issues + bonus fixes. Findings: (1) Regen cooldown init FIXED correctly — `gameState.js:81` now inits per-class via `REGEN_COOLDOWN[classKey]`, plus `combat.js:653` fallback guard. (2) Visual feedback FIXED — `postAttackMsg` in `combat.js:236-241` and DOT ticks at `combat.js:108-111,124-126` call `spawnDamageNumber`/`triggerShake`. (3) Wandering monsters added at `main.js:336-374`, every 25 turns, spawned off-screen with +20% stats. Addresses wait-to-win somewhat but regen cooldown being a finite window means patient players still fully heal BETWEEN wandering spawns. (4) Scroll of Fireball onKill FIXED (`items.js:211-213`) but STILL has no faction filter — hits ALL entities in radius 3, including the player. CombatSystem.aoeAttack has a faction filter but `_aoeFireball` bypasses it entirely. (5) Backstab rework: Evade grants stealth → Backstab does 3× from stealth, 1.5× otherwise. Two-turn setup in combat is awkward since Evade only lasts 1 turn and getting hit breaks stealth. (6) Score formula improved — `hud.js:525-534` uses `floor*1000 + level*200 + xp` with penalty only for excess turns. But `handleVictory` in `main.js:520` calculates `score * 2` using a DIFFERENT doubling than `hud.js:514` which also does `calculateScore(player) * 2`, resulting in the same number being shown AND saved. Harmless duplication but sloppy. (7) `floorTurns` not saved/loaded — `main.js:106` sets it, save/load in `main.js:534-620` doesn't serialize it. After loading, wandering monster timer resets to 0 = free turns. (8) `_aoeFireball` still bypasses `applyDamage()` — directly does `e.hp -= dmg`, so shields, evade, divine shield, and vulnerable status are all ignored. (9) Auto-ranged attack on movement STILL fires without meaningful feedback and adds `visibleTiles` fog check but could fire at targets the player didn't intend to engage.
- **Design Review #3 (2026-02-27)**: Fix verification review. 9 claimed fixes reviewed against source. Confirmed GOOD: (1) Fireball scroll now routes through `CombatSystem.aoeAttack` (`items.js:200-201`) — faction filter works, onKill works, applyDamage works. Major fix. (2) `floorTurns` reset on floor change (`main.js:416`) and incremented each turn (`main.js:202`). Save/load still doesn't serialize it, but `player.floorTurns` is on the player object which IS serialized — so this works by accident via entity save. (3) Arcane Shield duration 99→10 turns (`combat.js:474`) — reasonable. (4) Trap damage uses `CombatSystem.applyDamage` (`main.js:281`) — shields/evade work on traps now. (5) Unequip respects inventory cap (`items.js:618-623`) — drops to ground if full. (6) `dropLoot` uses deterministic seeded RNG (`items.js:716`). (7) Particles filtered by FOV (`renderer.js:302`). (8) Dead code removed — confirmed no fallback XP/levelUp paths in main.js. (9) Entity schema centralized in `gameState.js:83-84` with `stealthed`/`hasSeenPlayer`. STILL NOT FIXED: (A) Score doubling still duplicated — `main.js:525` does `calculateScore(player) * 2` and `hud.js:514` does `calculateScore(player) * 2`. Same result shown twice, but logic is copy-pasted in two places. (B) `hasSeenPlayer` backdoor on backstab still present — `combat.js:489` grants 3× if `!target.hasSeenPlayer`, meaning first Backstab on any unaware monster is always 3× even without Evade. (C) Auto-ranged still fires without confirmation (`main.js:259-276`). (D) Wait-to-win: 25-turn wandering interval with 5-8 turn regen cooldown means 17-20 turn dead gap where nothing threatens the player. Regen cooldown is a band-aid, not a solution.
